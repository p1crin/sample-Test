# テストケース登録 操作マニュアル

## 概要

本マニュアルでは、テスト管理システムにログインし、テストケースを新規登録するまでの操作手順を説明します。

---

## 前提条件

- ユーザアカウントが登録済みであること
- テストケースを登録するテストグループが作成済みであること
- 対象テストグループに対して**設計者（DESIGNER）**ロールが割り当てられていること

---

## 1. ログイン

1. ブラウザでシステムのURLにアクセスします。
2. ログイン画面が表示されます。

<!-- スクリーンショット: ログイン画面 -->

3. **メールアドレス**と**パスワード**を入力します。
4. 「ログイン」ボタンをクリックします。
5. ログインに成功すると、テストグループ一覧画面に遷移します。

> **補足:** パスワードを忘れた場合は、管理者にお問い合わせください。

---

## 2. テストグループの選択

1. テストグループ一覧画面から、テストケースを登録したいテストグループを選択します。

<!-- スクリーンショット: テストグループ一覧画面 -->

2. テストグループのテストケース一覧画面が表示されます。

<!-- スクリーンショット: テストケース一覧画面 -->

---

## 3. テストケース登録画面を開く

1. テストケース一覧画面で「テストケース登録」ボタンをクリックします。
2. テストケース登録画面が表示されます。

<!-- スクリーンショット: テストケース登録画面（全体） -->

---

## 4. テストケース情報の入力

以下の項目を入力します。

### 4.1 基本情報

| 項目 | 必須 | 説明 |
|------|------|------|
| TID（テストID） | ○ | テストケースの一意な識別子。同一テストグループ内で重複不可 |
| 第1層 | ○ | テスト分類（大分類） |
| 第2層 | ○ | テスト分類（中分類） |
| 第3層 | ○ | テスト分類（小分類） |
| 第4層 | ○ | テスト分類（詳細分類） |
| 目的 | ○ | テストの目的 |
| 要求ID | ○ | 関連する要求の識別子 |
| 確認項目 | ○ | テストで確認する項目 |
| テスト手順 | ○ | テストの実施手順 |

<!-- スクリーンショット: 基本情報入力エリア -->

> **注意:** TIDの入力欄からフォーカスが外れると、重複チェックが自動的に行われます。既に登録済みのTIDを入力した場合はエラーが表示されます。

### 4.2 添付ファイル（任意）

必要に応じて以下のファイルを添付できます。

| ファイル種別 | 説明 |
|-------------|------|
| 制御仕様書 | テストに関連する制御仕様書ファイル |
| データフロー | テストに関連するデータフロー図ファイル |

<!-- スクリーンショット: ファイル添付エリア -->

### 4.3 テスト内容の入力

テスト内容をテーブル形式で入力します。1つのテストケースに対して、複数のテスト内容を登録できます。

| 項目 | 説明 |
|------|------|
| テストケース | 具体的なテスト内容 |
| 期待値 | テスト実施時に期待される結果 |
| 対象 | テスト対象とするかどうか（チェックボックス） |

<!-- スクリーンショット: テスト内容テーブル -->

- 行の追加：「行追加」ボタンをクリックすると、新しい行が追加されます。
- 行の削除：対象行の「削除」ボタンをクリックすると、行が削除されます。

---

## 5. 登録の実行

1. すべての必須項目を入力したことを確認します。
2. 「登録」ボタンをクリックします。

<!-- スクリーンショット: 登録ボタン -->

3. 入力内容にエラーがある場合、該当箇所にエラーメッセージが表示されます。内容を修正して再度「登録」ボタンをクリックしてください。

<!-- スクリーンショット: バリデーションエラー例 -->

4. 登録が成功すると「テストケースを登録しました」というメッセージが表示されます。

<!-- スクリーンショット: 登録完了メッセージ -->

5. その後、テストケース一覧画面に自動的に遷移します。登録したテストケースが一覧に表示されていることを確認してください。

---

## 操作フロー図

```
ログイン画面
  │
  ├─ メールアドレス・パスワードを入力
  ├─ 「ログイン」をクリック
  │
  ▼
テストグループ一覧
  │
  ├─ 対象のテストグループを選択
  │
  ▼
テストケース一覧
  │
  ├─ 「テストケース登録」をクリック
  │
  ▼
テストケース登録画面
  │
  ├─ 基本情報を入力（TID、分類、目的など）
  ├─ 添付ファイルをアップロード（任意）
  ├─ テスト内容を入力
  ├─ 「登録」をクリック
  │
  ▼
登録完了 → テストケース一覧へ自動遷移
```

---

## トラブルシューティング

| 症状 | 対処方法 |
|------|---------|
| ログインできない | メールアドレスとパスワードが正しいか確認してください。解決しない場合は管理者にお問い合わせください |
| 「テストケース登録」ボタンが表示されない | 対象テストグループの設計者（DESIGNER）ロールが割り当てられていない可能性があります。管理者にロールの確認を依頼してください |
| TID重複エラーが表示される | 同じテストグループ内で既に使用されているTIDです。別のTIDを入力してください |
| 登録ボタン押下後にエラーが表示される | 必須項目の入力漏れがないか確認してください。エラーメッセージに従い修正してください |
